# Jest API Project

## Описание проекта
Проект представляет собой простое backend-приложение на Node.js с использованием Express.  
Цель проекта: продемонстрировать написание тестов с помощью Jest и Supertest.  

Приложение позволяет работать с пользователями через API:  

- Получение списка пользователей (`GET /users`)  
- Создание нового пользователя (`POST /users`)

---

## Структура проекта
jest-api-project/

│

├── src/                        # Исходный код приложения

│   ├── controllers/            # Контроллеры, обрабатывают HTTP-запросы

│   ├── routes/                 # Маршруты для API

│   ├── services/               # Логика работы с данными

│   └── utils/                  # Вспомогательные функции (например, валидация)

│

├── tests/                      # Тесты проекта

│   ├── unit/                   # Unit-тесты для отдельных функций

│   └── functional/             # Functional-тесты для проверки работы API

│

├── docs/test-results/          # Скриншоты результатов тестирования

├── package.json                # Список зависимостей и скриптов

└── README.md                   # Документация проекта

## Установка проекта

1. Клонируйте репозиторий или распакуйте архив проекта.  
2. Откройте проект в VS Code.  
3. В терминале VS Code убедитесь, что находитесь в корне проекта (`jest-api-project`).  
4. Установите зависимости командой:

```bash
npm install

Запуск сервера


Чтобы запустить сервер и проверить API:

npm start
Сервер будет доступен по адресу: http://localhost:3000

Проверка API

GET /users — получение списка пользователей (возвращает массив)
POST /users — создание пользователя. Пример запроса:

{

  "name": "Alex"
}

Ответ сервера:

{
  "id": 1,
  "name": "Alex"
}

Тестирование

Проект покрыт тестами с использованием Jest и Supertest.

Unit-тесты (10 тестов)

Unit-тесты проверяют отдельную функцию validator.js (валидация имени пользователя).

Проверяются следующие случаи:

обычное имя
пустая строка
строка из пробелов
числа, null, undefined
очень длинные строки (>50 символов)
граничные значения (50 символов)
специальные символы и пробелы


Файл тестов: tests/unit/validator.test.js


Functional-тесты (2 теста)

Functional-тесты проверяют работу всей системы:

GET /users — возвращает пустой массив при отсутствии пользователей
POST /users — создаёт пользователя и возвращает объект с id и name

Файл тестов: tests/functional/user.test.js

Тесты используют библиотеку Supertest для отправки HTTP-запросов к серверу.

Запуск тестов
В терминале VS Code введите:

npm test
Jest выполнит все unit и functional тесты, выводя результат в терминале.
